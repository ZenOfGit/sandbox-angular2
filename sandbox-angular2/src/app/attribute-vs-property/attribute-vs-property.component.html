We are not binding to the disabled attribute of the button element, the attribute is used to initialize the property.
We are actually binding to the disabled PROPERTY of the button object in the DOM once angular uses the attribute to crete it.


Notes:

  • Attributes are defined by HTML but properties are defined by the DOM
  • Attributes initialize DOM properties, so once the initialization is complete the attribute’s job is done, effectively disappearing after that
  • Property values can change, attribute value cannot
The following input element has a value attribute that will initialize the value property of this input object in the DOM to Guy
Using inputId.getAttribute('value') and inputId1.value in the console will show "Guy" as our current value.
Changing the name to "Bloke" will still show "Guy" if we use inputId.getAttribute('value') because that's the attribute.
However if we enter inputId1.value, this time it will show the DOM property has the value which we have changed to "Bloke".
