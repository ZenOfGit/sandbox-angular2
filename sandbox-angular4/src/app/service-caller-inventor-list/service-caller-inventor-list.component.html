A service in angular is used any time you need to reuse data or logic across multipole components.
The service used for these inventors contains hard coded data, but it would normally fetch it from an API.
As with anything we want to use in our components, we export the service class.
We have imported an interface for our inventors. Our component has imported the service.
We will inject our service into our controller's constructor. We have also listed the controller as a provider.
Service calls are usually issued over http to remote web servers, so delays may happen, so the best place for
for service calls is the ngOnInit lifecycle hook which executes after the constructor.
A class constructor is automatically called when an instance of the class is created, and is generally used to initialize the fields.
The ngOnInit lifecycle hook method is called after the constructor and is generally used to perform tasks related to data bindings.
